# World(=Scene) 운용 가이드

Rex에서는 월드가 별도 클래스가 아니라 `Scene`으로 표현된다.

## 1. 권장 업데이트 순서
1. 입력 처리
2. 명령 적용(UI/게임)
3. 물리 업데이트
4. 렌더링
5. UI 오버레이 렌더

## 2. 멀티 씬 운용
현재 Scene 전환은 사용자 코드에서 직접 관리한다.

권장 패턴:
- `std::unique_ptr<Scene> activeScene`
- 로드 완료 후 포인터 교체
- 시스템(Physics/UI) 내부 캐시 재동기화

## 3. 저장/복원 전략(권장)
기본 직렬화가 없으므로 다음 분리 기준을 권장한다.
- 엔티티 목록
- 컴포넌트 타입별 데이터
- 버전 메타/마이그레이션 정보

## 4. 에디터 관점
- Hierarchy = Scene 엔티티 목록
- Inspector = 선택 엔티티 컴포넌트 뷰
- Undo/Redo 도입 시 직접 수정 대신 Command 사용

## 5. 내부 개발자 체크리스트
- 씬 파괴 시 시스템 캐시 해제 여부
- 내부 포인터(`internalBody`) 유효성
- Scene 교체 시 UI 선택 상태 리셋 정책
